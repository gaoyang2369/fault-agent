# M0 范围定义

## 1. 产品目标

建立一个面向少量驱动电机/变频器设备的故障诊断 Agent。第一版支持：

- 查询 `G120-1` 的历史运行数据；
- 展示运行趋势和摘要；
- 查询故障码及厂家知识；
- 检查数据质量；
- 识别设备上报故障和报警；
- 识别少量确定性异常；
- 输出带证据的故障假设和检查建议；
- 生成简版运行诊断报告；
- 按角色限制数据和功能。

## 3. 第一版使用者

- 游客；
- 工程师；
- 管理员。

详细权限见 `02_personas_permissions.md`。

## 4. M0 Golden Path

> 用户查询 `G120-1` 的指定时间范围，系统读取 `real_data`，完成时间归一化和数据质量检查，识别故障码、报警码和运行异常，检索适用手册，形成证据化的诊断摘要，并可生成简版报告。

## 5. 第一版必须完成

- 资产和测点字典；
- `real_data` 只读适配器；
- 时间范围查询；
- 数据聚合和质量统计；
- 故障码/报警码归一化；
- 故障知识查询；
- Evidence、Claim、Hypothesis 数据结构；
- 简单规则框架；
- Agent 单任务和复合任务编排；
- 权限检查；
- 诊断摘要和简版报告；
- 可自动执行的验收场景。

## 6. 第一版明确不做

- 实时采集和流处理；
- 远程启停或修改变频器参数；
- 自动停机建议执行；
- 自动创建正式维修工单；
- 多租户；
- 多设备因果传播分析；
- 剩余寿命预测；
- 无监督模型直接上线；
- LLM 直接访问数据库；
- LLM 直接确认根因；
- 未经审核的知识文档自动发布；
- 把管理员视为无条件绕过所有审计的超级用户。

## 7. 成功标准

对于固定的测试数据、规则版本和知识版本：

1. 数据查询结果可重复；
2. 故障/报警事件识别可重复；
3. 每条关键诊断结论能够关联 Evidence；
4. 数据不足时不会强行下结论；
5. 未授权用户无法扩大时间范围或资产范围；
6. LLM 表述变化不影响结构化诊断结果；
7. 简版报告中的数字与结构化结果一致。
