# faultAgent M0：G120-1 驱动系统

版本：0.1.0  
状态：**可进入 M1（领域模型与契约设计）**  
适用范围：单机、非多租户、历史模拟数据、只读诊断

## 1. M0 结论

首个闭环场景定义为：

> 工程师或授权用户查询 `G120-1` 驱动系统指定时间范围内的运行数据；系统检查数据质量，识别设备上报的故障/报警以及可解释的运行异常，检索适用的故障手册，形成带证据的故障假设和检查建议，并可生成简版运行诊断报告。

## 2. 关键建模修正

`SINAMICS G120` 是变频器/驱动产品系列。项目内部暂将 `G120-1` 定义为一个 `DriveSystem`，至少包含：

- `Inverter`：SINAMICS G120，具体控制单元、功率模块、固件待确认；
- `Motor`：被驱动电机，具体型号和铭牌参数待确认。

不能把变频器故障码直接等同于电机根因，也不能把运行异常直接标记成“已确认故障”。

## 3. 第一版诊断原则

1. `fault_code` 和 `alarm_code` 优先作为设备上报事件处理。
2. 阈值规则必须可配置并记录版本。
3. 未取得准确设备型号、固件、铭牌和手册前，不启用安全相关的固定温度/电流/负载阈值。
4. 第一版可以产生：
   - 数据质量结论；
   - 设备上报故障/报警；
   - 确定性异常；
   - 系统故障假设；
   - 检查建议。
5. 第一版不能由 LLM 直接产生：
   - 已确认根因；
   - 工业阈值；
   - 诊断置信度；
   - 停机或控制指令。
6. `ConfirmedFault` 默认只能由工程师确认。设备上报故障码属于 `ReportedFaultEvent`，不等于根因确认。

## 4. 数据来源

- 表：`real_data`
- 数据类型：模拟历史运行数据
- 名义采样周期：约 3 秒
- 当前没有实时采集链路
- M0/M1 对该表只读访问

## 6. M0 完成判断

当前 M0 已达到“可开始工程实现”的程度，但以下工作仍会阻塞生产级诊断：

- G120 精确型号、控制单元、功率模块和固件；
- 电机型号、铭牌和温度传感器信息；
- 所有数值字段的单位及缩放；
- `status_word` / `control_word` 编码；
- `fault_code` / `alarm_code` 无事件值和格式；
- 厂家手册及其适用版本；
- 一段经确认的健康基线数据。

这些问题不阻塞搭建后端骨架、数据适配器、权限、查询、证据模型和 Agent 编排。
