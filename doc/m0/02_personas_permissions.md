# 角色与权限

## 1. 权限模型

第一版不需要多租户，但仍需同时检查：

```text
角色权限
+ 资产范围
+ 时间范围
+ 操作类型
+ 数据粒度
```

角色只决定能力上限，资产分配决定实际可访问对象。

## 2. 权限矩阵

| 能力 | 游客 | 工程师 | 管理员 |
|---|---:|---:|---:|
| 查询公开故障码 | 允许 | 允许 | 允许 |
| 查看允许展示的设备 | 白名单 | 已分配设备 | 所有本地设备 |
| 查询历史数据 | 最近 1 小时 | 授权范围 | 全部 |
| 查看原始 3 秒数据 | 默认不允许 | 允许 | 允许 |
| 查看聚合数据 | 允许，建议 1 分钟聚合 | 允许 | 允许 |
| 执行故障诊断 | 不允许 | 允许 | 允许 |
| 生成简版报告 | 不允许 | 允许 | 允许 |
| 确认最终故障 | 不允许 | 允许，需记录身份 | 允许，需记录身份 |
| 上传知识文档 | 不允许 | 不允许 | 允许 |
| 发布知识版本 | 不允许 | 不允许 | 允许，必须审计 |
| 管理设备 | 不允许 | 不允许 | 允许 |
| 管理用户与资产分配 | 不允许 | 不允许 | 允许 |
| 修改诊断规则 | 不允许 | 不允许 | 第一版建议暂不开放 |
| 控制设备 | 不允许 | 不允许 | 不允许 |

## 3. 游客限制

- 只能访问配置在 `guest_visible_assets` 中的设备；
- 查询结束时间不得晚于当前数据最新时间；
- 查询跨度最多一小时；
- 返回 1 分钟或更粗粒度的聚合数据；
- 不返回敏感原始状态字和控制字；
- 不允许通过 Agent 组合请求绕过限制。

## 4. 工程师权限

- 只访问 `engineer_asset_assignments` 中的设备；
- 可查询原始数据和聚合数据；
- 可创建 DiagnosisRun；
- 可生成非正式简版报告；
- 可对系统假设进行确认、否定和备注；
- 不能上传或发布知识；
- 不能修改规则。

## 5. 管理员权限

- 管理本地所有资产和用户分配；
- 上传知识文档；
- 设置文档适用设备、变频器型号和固件版本；
- 发布或下架知识版本；
- 查看审计记录；
- 第一版仍不能执行设备控制。

## 6. 安全约束

- `user_id`、角色和资产范围来自认证上下文，不接受 LLM 参数；
- 每一次 Tool 调用重新鉴权；
- 后端必须再次检查游客一小时时间限制；
- Agent 只获得用户当前有权使用的工具；
- 知识上传和发布是两个不同权限；
- 所有诊断、报告、知识变更和故障确认必须审计。
