# 第一版诊断政策

## 1. 诊断分层

```text
L0 数据存在性和质量
L1 设备上报的故障/报警
L2 可配置的确定性异常规则
L3 健康基线偏离
L4 故障假设融合
L5 人工确认
```

M0/M1 优先实现 L0、L1 和规则框架。L2 只启用已确认规则；L3 需要健康基线；L4 第一版采用规则贡献和知识映射；L5 由工程师执行。

## 2. 故障码优先还是阈值优先

答案是：两者都需要，但语义不同。

### 故障码

- 是设备/变频器已经报告的事件；
- 通常具有明确产品和固件语义；
- 应优先精确查询；
- 可以确认“设备报告了某故障”；
- 不能单独确认物理根因。

### 阈值和行为规则

- 可以发现故障码出现前的异常；
- 可以发现传感器、跟踪误差、热趋势和负载问题；
- 必须结合工况、单位、铭牌和持续时间；
- 没有准确信息时只产生 `Anomaly`，不能产生 `ConfirmedFault`。

## 3. 第一版推荐策略

### 3.1 立即启用：数据质量规则

- 时间无法解析；
- 数据为空；
- 缺失率；
- 采样间隔异常；
- 大量重复；
- 数值字段全为空；
- 传感器长时间完全不变；
- 数据时间范围与请求不重叠。

### 3.2 在数据摸底后启用：设备事件规则

```text
fault_code 有效 → DEVICE_FAULT_REPORTED
alarm_code 有效 → DEVICE_ALARM_REPORTED
status 发生已知故障状态变化 → STATUS_FAULT_REPORTED
```

前提是确认无事件值和编码。

### 3.3 暂不硬编码：温度/电流/负载绝对阈值

在以下信息齐全前，不启用生产级固定阈值：

- 电机型号及铭牌；
- 温度测量对象；
- 绝缘等级；
- 变频器精确型号；
- 固件；
- 电机冷却方式；
- 厂家参数设置；
- 单位和缩放。

可以把阈值配置结构先建好，但默认 `enabled: false`。

### 3.4 推荐首批行为异常

在字段语义和单位确认后，可优先实现：

1. **速度跟踪误差**
   - `abs(speed_actual - speed_setpoint)`；
   - 只在运行状态和设定值超过最小门限时判断；
   - 需配置绝对/相对误差与持续时间。

2. **转矩跟踪误差**
   - `torque_actual` 与 `torque_setpoint`；
   - 需考虑加减速阶段。

3. **温升异常**
   - `motor_temp - air_intake_temp`；
   - 比单独绝对温度更适合早期趋势观察；
   - 仍需要健康基线或厂家限制。

4. **散热关联异常**
   - `inverter_temp`、`inverter_radiator_temp` 与 `air_intake_temp` 的关系；
   - 只产生“冷却/热管理异常假设”。

5. **持续高负载**
   - `motor_load_rate`、`inverter_load_rate`；
   - 确认百分比缩放后再启用。

6. **信号冻结**
   - 运行状态变化但关键字段长时间完全相同；
   - 产生传感器或采集链路异常。

7. **功率/速度/转矩一致性**
   - 需要明确字段定义和方向约定后再启用。

## 4. 健康基线

取得一段由工程师确认的正常运行数据后，建立按工况分组的基线：

- 温度中位数和波动；
- 速度跟踪误差；
- 电流、转矩、功率分布；
- 温升与负载关系；
- 启动、稳态、停机阶段特征。

统计偏离只产生 `BASELINE_DEVIATION`，不能自动确认根因。

## 5. LLM 的角色

允许：

- 解析请求；
- 调用查询、故障码和诊断工具；
- 将结构化结果转成说明；
- 在多个 Hypothesis 间说明支持/反证；
- 生成简版报告文字。

不允许：

- 根据直觉确定最终故障；
- 自报“90% 置信度”；
- 修改诊断结构化结果；
- 编造厂家手册内容；
- 用自然语言兜底掩盖缺失数据。

## 6. 第一版故障确认

状态建议：

```text
SYSTEM_ASSESSMENT
ENGINEER_REVIEW_REQUIRED
ENGINEER_CONFIRMED
ENGINEER_REJECTED
```

默认流程：

```text
系统生成 Hypothesis
→ 工程师查看证据
→ 工程师确认/否定/补充观察
→ 创建 ConfirmedFault（可选）
```

## 7. 诊断结果状态

- `NO_DATA`
- `INSUFFICIENT_DATA`
- `NO_DEVICE_EVENT_OR_ENABLED_ANOMALY`
- `DEVICE_ALARM_REPORTED`
- `DEVICE_FAULT_REPORTED`
- `ANOMALY_DETECTED`
- `HYPOTHESES_AVAILABLE`
- `PARTIALLY_COMPLETED`
- `FAILED`

## 8. 简版报告

第一版报告标题建议使用：

> AI 辅助运行诊断摘要

而不是“最终故障报告”。内容包括：

- 资产；
- 时间范围；
- 数据质量；
- 运行摘要；
- 故障/报警事件；
- 异常；
- 故障假设；
- 证据；
- 未确认事项；
- 建议检查；
- 规则、知识和模型版本。
